Audit basis:

- `FAIL` means the dimension has at least one release-blocking gap; `WARN` means the dimension is usable but should be remediated before or soon after release; `PASS` means no release-blocking issue was found in that dimension.

## Tier 1 -- Hard Blockers

1. **No OSS license file at repository root**
   - Evidence:
     - Missing `/Users/dawid/code/glosa/LICENSE`
     - Missing `/Users/dawid/code/glosa/LICENSE.md`
     - License scan reports root package as `UNLICENSED`
     - `/Users/dawid/code/glosa/package.json:4` has `"private": true`
   - Impact:
     - Public users would have no legal permission to use, modify, or redistribute the code.
   - Concrete remediation:
     1. Choose the intended project license.
     2. Add the full license text at `/Users/dawid/code/glosa/LICENSE`.
     3. Add the same SPDX identifier to `/Users/dawid/code/glosa/package.json` and workspace package manifests.
     4. Re-run license scanning to confirm no unknown/incompatible licenses remain.

2. **No public-facing repository README at root**
   - Evidence:
     - Missing `/Users/dawid/code/glosa/README.md`
     - Only `/Users/dawid/code/glosa/docs/README.md` exists, which is design-doc navigation, not a release README.
   - Impact:
     - For this audit, this is a release blocker because a public repository needs a minimum entrypoint explaining what the project is, how to install/use it, its stability status, and the license users receive.
   - Concrete remediation:
     1. Create `/Users/dawid/code/glosa/README.md`.
     2. Include:
        - what glosa is
        - install/setup steps
        - minimal usage example
        - project status/stability note
        - license identifier/link
     3. Keep `/Users/dawid/code/glosa/docs/README.md` as internal design navigation if useful.

3. **No branch protection or rulesets on `main`**
   - Evidence:
     - `gh api repos/davebream/glosa/branches/main/protection` returned “Branch not protected”
     - `gh api repos/davebream/glosa/rulesets` returned `[]`
   - Impact:
     - For this audit, this is a release blocker because a public default branch should be protected from accidental force pushes, deletion, and unreviewed changes.
   - Concrete remediation:
     1. In GitHub repository settings for `davebream/glosa`, configure branch protection or repository rulesets targeting the `main` branch.
     2. At minimum require:
        - pull request before merge
        - blocked force pushes
        - blocked branch deletion
        - required status checks
     3. If you want direct-to-main in day-to-day work, still configure baseline protection appropriate for a public repo.
